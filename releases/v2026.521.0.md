# v2026.521.0

> Released: 2026-05-21

## Highlights

- **Runtime language guidance is more reliable** - Paperclip now injects a compact, high-priority runtime language note across local and gateway adapters, making agent-facing responses more likely to follow the configured runtime language while preserving code, paths, logs, API names, and user-provided text. ([#99](https://github.com/penclipai/paperclip-cn/pull/99), @penclipai)
- **Provider vault setup is easier to operate** - The Secrets area now supports provider vault workflows, including AWS Secrets Manager discovery, provider-backed secret selection, clearer validation, and localized UI coverage. ([#6381](https://github.com/paperclipai/paperclip/pull/6381), @cryppadotta)
- **Workspace diff review is more polished** - The first-party workspace diff plugin has better review ergonomics, stronger worker behavior, and expanded tests for inspecting workspace changes inside Paperclip. ([#6383](https://github.com/paperclipai/paperclip/pull/6383), @cryppadotta)

## Improvements

- **Agent prompt guidance stays compact** - The runtime prompt now explicitly prefers the `penclip` CLI for Paperclip operations, avoids lossy direct POST payloads when a CLI path exists, and asks agents to create filenames with UTF-8/Unicode-safe names. ([#99](https://github.com/penclipai/paperclip-cn/pull/99), @penclipai)
- **Developer and operator surfaces are clearer** - Dev restart status, sidebar behavior, inbox rows, markdown editing, and recovery action displays received quality-of-life updates and broader tests. ([#6384](https://github.com/paperclipai/paperclip/pull/6384), @cryppadotta)
- **Cloud tenant imports are less brittle** - Cloud tenant import mutations can proceed without browser origin headers, while large import payloads use a dedicated portable package body limit. ([#6378](https://github.com/paperclipai/paperclip/pull/6378), [#6380](https://github.com/paperclipai/paperclip/pull/6380), @cryppadotta)
- **CN fork localization remains aligned** - Recovery action surfaces and duplicate-agent actions keep zh-CN coverage while upstream runtime changes are absorbed into the fork. ([#96](https://github.com/penclipai/paperclip-cn/pull/96), [#98](https://github.com/penclipai/paperclip-cn/pull/98), [#99](https://github.com/penclipai/paperclip-cn/pull/99), @penclipai)
- **Release publishing follows trusted publishing** - The fork release workflow now relies on GitHub OIDC trusted publishing instead of the stale npm token path, unblocking canary and latest promotion. ([#101](https://github.com/penclipai/paperclip-cn/pull/101), @penclipai)

## Fixes

- **Runtime control-plane edge cases are handled more safely** - Feedback export flushing now backs off when the database is unavailable, plugin lifecycle restarts are cleaner, and issue thread confirmations can be expired by later human comments. ([#6380](https://github.com/paperclipai/paperclip/pull/6380), @cryppadotta)
- **Database conflict retries survive wrapped driver errors** - Conflict retry detection continues to work when Drizzle wraps the underlying database error. ([#97](https://github.com/penclipai/paperclip-cn/pull/97), @penclipai)
- **Recovery work uses cheaper models more carefully** - Recovery model hints better guard cheap-model usage for cases that require stronger reasoning. ([#6371](https://github.com/paperclipai/paperclip/pull/6371), @cryppadotta)
- **Sandbox-provider plugins stay out of instance settings** - Sandbox-provider plugins are hidden from the Instance Settings sidebar where they do not belong. ([#6341](https://github.com/paperclipai/paperclip/pull/6341), @devinfoley)

## Upgrade Guide

- No manual migration is required beyond the normal Paperclip upgrade path.
- Operators using runtime default language settings should see more consistent agent-facing natural-language responses, but technical identifiers, commands, paths, logs, and user-authored text remain intentionally untranslated.
- Maintainers should continue using the fork release workflow so stable publishing runs from the tested `master` source ref and attaches the `releases/v2026.521.0.md` notes to the GitHub Release.
