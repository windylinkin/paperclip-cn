# v2026.519.0

> Released: 2026-05-19

## Highlights

- **Workspace changes are easier to review** - A new first-party workspace diff plugin adds a Git-powered Changes tab for project and execution workspaces, helping operators inspect file changes without leaving Paperclip. ([#94](https://github.com/penclipai/paperclip-cn/pull/94), @penclipai)
- **Modal joins the sandbox provider lineup** - Paperclip can now load the Modal sandbox provider as an external plugin package, with plugin manifests, runtime hooks, configuration validation, tests, and release enrollment in place. ([#94](https://github.com/penclipai/paperclip-cn/pull/94), @penclipai)
- **Secret-backed plugin configuration is more ergonomic** - JSON schema forms now understand secret-reference fields, so plugin and runtime configuration can select existing secrets or paste raw values with clearer validation and localized labels. ([#6339](https://github.com/paperclipai/paperclip/pull/6339), @devinfoley)
- **Paperclip CN keeps expanding localized UI coverage** - The fork now carries broader zh-CN coverage across issue workflows, plugin tabs, workspace configuration, generated notices, and multilingual issue text preservation. ([#94](https://github.com/penclipai/paperclip-cn/pull/94), @penclipai)

## Improvements

- **Grok Build is available as a local adapter** - The fork includes the upstream Grok local adapter work and has bootstrapped `@penclipai/adapter-grok-local` for future automated releases. ([#6087](https://github.com/paperclipai/paperclip/pull/6087), @devinfoley)
- **Issue documents are safer for handoffs** - Board-managed document locks, live document refresh, and restored issue document activity make issue handoffs more predictable during agent and human collaboration. ([#6009](https://github.com/paperclipai/paperclip/pull/6009), [#6005](https://github.com/paperclipai/paperclip/pull/6005), @cryppadotta)
- **High-volume issue boards are easier to scan** - Dense board lanes, collapsed cold columns, per-column reveal limits, and saved density controls improve large-company issue triage. ([#5309](https://github.com/paperclipai/paperclip/pull/5309), @eibrahim)
- **Routine runtime secrets have stronger contracts** - Routine environment secrets and runtime metadata now have schema, migration, and UI support, improving how recurring work receives sensitive configuration.
- **Plugin authoring is clearer** - The plugin SDK and docs now cover managed resources, UI bridge surfaces, workspace access, testing helpers, and compatibility import expectations for external plugin packages.
- **Release automation is more robust** - PR verification is more parallelized, release package bootstrap checks catch missing first publishes, and standalone sandbox-provider packages build their own publish payloads before npm staging.

## Fixes

- **Worktree initialization no longer wipes repository worktrees** - `worktree init --force` avoids deleting existing repository worktree directories. ([#6240](https://github.com/paperclipai/paperclip/pull/6240), @devinfoley)
- **New issue autocomplete is clickable again** - Pointer selection in the new issue dialog works correctly when choosing suggestions. ([#6311](https://github.com/paperclipai/paperclip/pull/6311), @penclipai)
- **Secret form textarea layout is more resilient** - Long secret values and narrow layouts no longer overflow the new secret form. ([#6222](https://github.com/paperclipai/paperclip/pull/6222), @devinfoley)
- **Sandbox cold starts have more room to finish** - Plugin environment probes now allow longer RPC timeouts for cold-start sandboxes, reducing false failures during startup. ([#6289](https://github.com/paperclipai/paperclip/pull/6289), @devinfoley)
- **Grok reasoning transcripts regain readable boundaries** - Streamed Grok reasoning chunks preserve turn separation instead of merging into one run-on transcript. ([#6142](https://github.com/paperclipai/paperclip/pull/6142), @devinfoley)

## Upgrade Guide

- No manual migration is required beyond the normal Paperclip upgrade path. The release includes database migrations for issue documents and routine runtime secret metadata; let the server run its standard migration flow on first boot.
- Operators using npm-installed plugins can now install `@penclipai/plugin-modal` once Modal credentials are configured. The package has been bootstrapped on npm under `latest`.
- Maintainers should finish npm trusted-publishing setup for newly bootstrapped `@penclipai/adapter-grok-local` and `@penclipai/plugin-modal` before relying on future automated stable publishes for those packages.
