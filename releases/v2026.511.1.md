# v2026.511.1

> Released: 2026-05-11

## Highlights

- **Safer remote sandbox startup** - Local adapters now get a longer default timeout when running inside remote sandboxes, and sandbox install probes honor adapter timeout overrides so installs and warm-up steps have enough time to complete. ([#5685](https://github.com/paperclipai/paperclip/pull/5685), [#90](https://github.com/penclipai/paperclip-cn/pull/90))
- **Non-lossy routine scheduling** - Routine triggers now use a card-and-dialog flow, and the schedule editor preserves common cron shapes such as multiple daily times, every-N-minute windows, every-N-hour schedules, selected weekdays, and monthly date lists. ([#3569](https://github.com/paperclipai/paperclip/pull/3569), [#90](https://github.com/penclipai/paperclip-cn/pull/90))
- **Stronger CN fork localization maintenance** - Issue, goal, system-notice, related-work, and productivity-review surfaces gained additional zh-CN coverage, and the fork now includes a reusable i18n key snapshot audit for future upstream syncs. ([#90](https://github.com/penclipai/paperclip-cn/pull/90))

## Improvements

- **Cursor sandbox installs** - The Cursor adapter now resolves `cursor-agent` from the installed Cursor binary directory inside remote sandboxes instead of relying on PATH assumptions. ([#5686](https://github.com/paperclipai/paperclip/pull/5686), [#90](https://github.com/penclipai/paperclip-cn/pull/90))
- **Adapter sandbox install fallback** - Adapter install commands can fall back to a writable `$HOME/.local` prefix when global npm install locations are unavailable in sandbox environments. ([#5685](https://github.com/paperclipai/paperclip/pull/5685), [#90](https://github.com/penclipai/paperclip-cn/pull/90))
- **Fork maintenance runbooks** - Release, Docker, upstream merge, and UI localization docs were refreshed to reflect embedded PostgreSQL, release post-checks, and the long-term sync audit surfaces that matter for this fork. ([#90](https://github.com/penclipai/paperclip-cn/pull/90))
- **Translation key audit workflow** - The `paperclip-cn-fork-maintenance` skill now records how to snapshot and compare locale keys before and after upstream syncs, including explicit cautions for dynamic keys and runtime/plugin metadata. ([#90](https://github.com/penclipai/paperclip-cn/pull/90))

## Fixes

- **Codex and Claude remote probes** - Remote adapter probes now use safer probe flags and permission handling when checking command availability outside normal local repository contexts. ([#5685](https://github.com/paperclipai/paperclip/pull/5685), [#90](https://github.com/penclipai/paperclip-cn/pull/90))
- **Windows sandbox test paths** - Cursor sandbox tests preserve remote paths as POSIX-style values and convert back to Windows paths only at the local runner boundary. ([#90](https://github.com/penclipai/paperclip-cn/pull/90))
- **Locale catalog hygiene** - High-confidence unused locale keys were removed from both English and zh-CN catalogs after static usage checks, exact reference searches, and locale test coverage. ([#90](https://github.com/penclipai/paperclip-cn/pull/90))

## Upgrade Guide

- Paperclip will apply any included database migrations on normal startup. No manual SQL step is expected for this release.
- If you run adapters in remote sandboxes, allow the next canary/stable install to refresh adapter packages so the new timeout and install-resolution behavior is available.
- If you maintain CN fork translations, run the new i18n key audit before and after future upstream syncs, but do not treat `unusedKeys` as automatic deletion authority.
