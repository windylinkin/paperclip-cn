# v2026.511.0

> Released: 2026-05-11

## Highlights

- **Secrets provider vaults and imports** - Company settings now support provider-backed secrets, remote import flows, access auditing, and richer secret binding UI so operators can bring managed credentials into Paperclip without copying values by hand. ([#5429](https://github.com/paperclipai/paperclip/pull/5429))
- **Cursor Cloud adapter** - Paperclip can run work through the Cursor SDK and Cloud Agents API v1, with package metadata and UI registration preserved under the Paperclip CN package scope. ([#5664](https://github.com/paperclipai/paperclip/pull/5664), [#85](https://github.com/penclipai/paperclip-cn/pull/85))
- **Plugin host and managed skills groundwork** - The plugin host now has broader SDK, managed-skill, managed-agent, and local-folder support, giving future plugins a clearer path to ship UI/runtime behavior through generic host surfaces. ([#5597](https://github.com/paperclipai/paperclip/pull/5597), [#85](https://github.com/penclipai/paperclip-cn/pull/85))
- **Sandbox provider expansion** - Daytona joins the sandbox provider plugin set, and the Paperclip CN fork bootstrapped `@penclipai/plugin-daytona` for normal CI release publishing. ([#5580](https://github.com/paperclipai/paperclip/pull/5580), [#85](https://github.com/penclipai/paperclip-cn/pull/85))

## Improvements

- **Paperclip CN upstream sync** - The fork absorbed upstream through `6e4fa78d` while preserving zh-CN locale coverage, `@penclipai/*` package names, `penclip` command wording, Windows-sensitive behavior, Electron boundaries, and external adapter discipline. ([#85](https://github.com/penclipai/paperclip-cn/pull/85))
- **Runtime target configuration** - Remote execution target resolution now carries the environment id through sandbox and SSH runtime config paths, preserving secret-backed environment behavior after the upstream sync. ([#85](https://github.com/penclipai/paperclip-cn/pull/85))
- **Localization coverage** - Live dashboard copy, issue chat fallback text, JSON schema form controls and errors, plugin/environment schema descriptions, and merge-touched UI surfaces now have refreshed English and zh-CN coverage. ([#85](https://github.com/penclipai/paperclip-cn/pull/85))
- **Release package reliability** - Release registry verification is more tolerant of npm propagation lag, Daytona packaging rewrites local compatibility aliases to public `@penclipai/*` npm aliases, and release-enabled package enrollment is explicit. ([#4816](https://github.com/paperclipai/paperclip/pull/4816), [#85](https://github.com/penclipai/paperclip-cn/pull/85))

## Fixes

- **Docker image build** - Removed a stale Dockerfile manifest copy for a non-existent LLM Wiki package so the container deps stage can build from the actual workspace package set. ([#86](https://github.com/penclipai/paperclip-cn/pull/86))
- **Daytona Windows compatibility** - Daytona plugin scripts now use Windows-safe command surfaces while keeping source imports compatible with the upstream plugin SDK package name. ([#85](https://github.com/penclipai/paperclip-cn/pull/85))
- **Registry checks** - Canary registry verification retries and stale metadata handling were strengthened to reduce false negatives during publish verification. ([#5579](https://github.com/paperclipai/paperclip/pull/5579), [#4816](https://github.com/paperclipai/paperclip/pull/4816))

## Upgrade Guide

- Paperclip will apply the included database migrations on normal startup. No manual SQL step is expected.
- If you use secrets providers, review the new company secrets settings and provider configuration before moving sensitive values into managed vault imports.
- If you use Cursor Cloud, configure the required Cursor credentials and cloud-agent access before assigning production work to the `cursor_cloud` adapter.
- Daytona is now enrolled in normal Paperclip CN release publishing after the one-time `@penclipai/plugin-daytona@0.1.0` bootstrap publish.
- Existing GitHub and HTTP(S) project remotes continue to work. SSH-style Git remotes remain outside this release.
